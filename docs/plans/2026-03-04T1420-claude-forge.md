## Agent Design Decisions

### 1. Self-contained (no Include:)
Claude Code doesn't support `Include:` directives. All agent system prompts are inlined. VPS `.opencode/` agents use `Include:` to reference ROLE spec files — Claude Code agents do not.

### 2. Stage gates are constitutional
A-AUDITOR non-SEAL → blocks A-ENGINEER start (no bypass).
A-VALIDATOR non-SEAL → blocks stage 999 (no bypass).
A-ORCHESTRATOR enforces this; never silently skips.

### 3. Tool restrictions by role
- A-ARCHITECT: read-only (no Bash, no Write/Edit)
- A-AUDITOR: read-only + Bash (scan only, no edit)
- A-VALIDATOR: read-only + Bash (verify, no edit)
- A-ENGINEER: full tools (Read/Write/Edit/Bash)
- A-ORCHESTRATOR: Agent tool (spawns subagents) + no direct edit

### 4. aaa-governance skill = Gen3 canonical
Includes full Gen1→Gen3 tool name translation table. Any agent using this skill cannot accidentally call deprecated tool names.
